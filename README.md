# Muse - AI 音乐创作 Skill

> 凌晨两点，你突然有了一段旋律的灵感。
> 打开终端，输入一句话 —「深夜独自开车回家的孤独感」。
> 2 分钟后，一首完整的歌回来了。带人声，带伴奏，甚至还有封面。
>
> 这不是未来，这是 Muse。

## 什么是 Muse

Muse 是一个开源的 AI 音乐创作 Skill。你不需要懂乐理，不需要会编曲，只需要一句话描述你想要的歌，AI 帮你从词到曲到人声全部搞定。

**免费使用** — 注册即送创作积分，够你做几十首歌，不花一分钱就能体验完整功能。

**稳定可靠** — 底层对接专业 AI 音乐引擎，经过完整的错误处理和重试机制打磨，Token 过期自动续、网络抖动自动兜底、积分不足提前拦截。不会半路崩掉让你干等。

**开箱即用** — Python 脚本零外部依赖（只用标准库），不需要 `pip install` 任何东西。安装一行命令，30 秒注册，第三分钟你就在听自己的歌了。

## 三种创作模式

| 模式 | 你做什么 | AI 做什么 | 适合场景 |
|------|---------|----------|---------|
| **灵感模式** | 一句话描述 | 选风格、编词、谱曲、演唱 | 灵感来了，不想多想 |
| **定制模式** | 选风格、写歌词 | 严格执行你的要求 | 精雕细琢，每个字都讲究 |
| **纯音乐** | 描述氛围 | 生成无人声伴奏 | 视频 BGM、播客配乐 |

支持 115 种音乐风格标签 — 流行、摇滚、民谣、古风、电子、爵士、说唱、R&B...你想得到的都有。

## 安全与隐私

> Muse 将用户数据安全和隐私保护作为最高优先级。以下为完整的安全架构说明。

### 零依赖架构 — 最小化攻击面

Muse 的脚本层仅使用 **Python 标准库**（`urllib`、`json`、`hashlib`、`uuid`、`os`、`platform`），**不引入任何第三方依赖**。

这意味着：
- **无供应链攻击风险** — 没有 `pip install`，没有 `node_modules`，不存在恶意包注入的可能性
- **无 CVE 暴露面** — 不依赖第三方库版本，不受上游漏洞影响
- **完整可审计** — 全部源码约 300 行 Python，任何人可在 10 分钟内完成全量代码审计

> **OWASP A06:2021 — Vulnerable and Outdated Components**
> 零依赖设计从根本上消除了此类风险。

### 登录与账号安全

| 保护措施 | 说明 |
|----------|------|
| **手机验证码登录** | 每次登录都需要手机接收短信验证码，只有你本人能操作 |
| **设备绑定** | 自动识别你的电脑，换一台设备需要重新验证，别人拿到密码也登不了你的号 |
| **登录凭证只存本地** | 你的登录信息只保存在自己电脑上，不会上传到任何地方 |
| **全程加密传输** | 所有通信都经过银行级加密通道（HTTPS），第三方无法窃听 |
| **过期自动注销** | 登录状态过期后自动清除，不会残留旧凭证 |
| **权限最小化** | 登录凭证只能用来做歌，无法访问你的文件、通讯录或其他任何东西 |

### 隐私保护

**Muse 只在你的电脑上保存两样东西：** 登录凭证和设备标识 — 仅此而已。

- **不偷看** — 不会记录你输入的歌词、描述或任何创作内容
- **不偷传** — 没有任何后台数据上报、统计埋点、用户画像采集
- **不偷存** — 生成的歌曲通过链接在线收听，不会在电脑里偷偷缓存
- **只传必要信息** — 和服务器通信时，只发送歌曲描述和风格选择，不携带任何个人信息

### 输入保护

你输入的每一条信息都会经过安全检查：

- 手机号格式不对？直接拦截，不会发出去
- 歌曲描述太短？提前提醒，避免白等一场
- 所有参数都有固定格式约束，无法被恶意利用

### 出错了怎么办

- 网络不好、服务器忙、积分不够 — 都会给你**明确的中文提示**，告诉你发生了什么、该怎么办
- 错误信息**不会暴露**你的个人数据或系统内部信息
- 不会出现看不懂的报错代码

### 防滥用机制

| 机制 | 说明 |
|------|------|
| **积分系统** | 每首歌消耗 10-15 积分，余额不足时提前告知，不会莫名扣费 |
| **一次只做一首** | 同一时间只能创作一首歌，防止被恶意批量刷取 |
| **频率保护** | 操作太频繁会自动限速，保障所有用户的服务质量 |
| **超时兜底** | 每个请求都有时间上限，不会无限等待卡死 |

### 代码完全公开

| 项目 | 说明 |
|------|------|
| **开源许可** | MIT 协议 — 任何人都可以免费使用、修改、分发 |
| **源码公开** | 所有代码完全开源，没有加密、没有隐藏、没有"看不到的部分" |
| **体量极小** | 总共约 300 行代码，懂编程的朋友 10 分钟就能通读一遍 |
| **无后门** | 不包含任何远程控制、后台执行、隐藏功能 |

## 安装

### 方式一：npm

```bash
npx muse-music-skill
```

### 方式二：Git Clone

```bash
git clone https://github.com/AI-Music-Muse/skills.git
cd skills
bash install.sh
```

安装完成，在对话中发送 **「做首歌」** 就行了。

## 兼容性

Muse 不绑定任何特定模型。本质上是 Markdown 提示词 + Python 脚本，任何能读指令、能跑 bash 的 AI CLI 都能用：

- Claude Code
- Kimi
- 千问
- 智谱
- ...

## 前置要求

- 任意支持 Skills 的 AI CLI
- Python 3.6+（仅标准库，零依赖）

## 技术架构

```
AI CLI ←→ SKILL.md（交互定义）
              ↓
          Python 脚本层（仅标准库，零依赖）
          ├── muse_api.py    API 封装
          ├── poll_song.py   异步轮询
          └── register.py    注册流程
              ↓
          skill-api.muse.top（中间件 API）
              ↓
          AI 音乐引擎
```
